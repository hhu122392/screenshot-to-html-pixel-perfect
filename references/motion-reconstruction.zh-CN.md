# 动效重建与逐帧审计

## 源时间轴

GIF 解码全部帧。录屏先用明确的开始和结束毫秒定义每个有效片段，再解码片段内全部帧。同时记录声明帧数和解码帧数；无法解释的漏帧直接失败。

## 动效账本

对每个运动主体记录开始/结束边界、变换中心、位置、缩放、旋转、透明度、层级、触发器、延迟、时长、缓动证据、循环规则和结束状态，并写入中英双语动效时间线与 `MOTION_TIMELINE.json`。

## 可控审计时钟

页面提供：

```javascript
window.__setAuditTime = (milliseconds) => {
  // 在准确相对时间渲染所有运动主体。
};
```

这个接口必须更新实时播放使用的同一套状态，禁止单独生成一张只用于审计的假画面。

## 逐帧截图和比较

1. 提取完整源帧序列。
2. 使用 `capture_interaction_frames.cjs` 按每个源时间点捕获候选帧。
3. 运行 `compare_frame_sequence.py`。
4. 要求帧数和尺寸一致。
5. 任意帧平均绝对通道差 `>= 2` 时修正实现。

记录交互锚点，保证同一个点击、触摸或滚动事件启动两条时间轴。
