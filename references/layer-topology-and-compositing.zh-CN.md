# 图层拓扑与透明合成

目标包含重叠图层、凸出插画、半透明外沿、阴影、模糊、抠图、凹形轮廓，或者组件像素受到未实现背景影响时，实现前阅读本文档。

## 证据充分性

给每个透明或依赖背景的区域分类：

- `recoverable`：源材料提供前景、Alpha、背景，或者提供了足够多的独立观察，可以还原。
- `source_composited`：截图中的前景已经和已知或可见背景混合。
- `non_identifiable`：前景色、Alpha 或被遮挡背景未知，无法从单张截图唯一反推原始图层。

透明合成公式是 `C = alpha * F + (1 - alpha) * B`。只有一个合成结果 `C`，无法同时求出未知的 `F`、`alpha` 和 `B`。写代码前登记这个限制。`non_identifiable` 区域不能承诺严格像素通过；取得透明源图、设计导出或足够多背景证据前，严格 1:1 交付必须保持失败。

证据限制 ID 必须和 `AUDIT_MAP.json` 中对应区域 ID 相同：

```json
{
  "id": "translucent-outer-rim",
  "evidence_class": "non_identifiable",
  "reason": "前景 Alpha 和被遮挡背景都没有独立证据。",
  "strict_pixel_claim_allowed": false
}
```

## 必需视觉模型

写页面代码前根据模板创建 `VISUAL_MODEL.json`，登记：

- 每个 DOM、SVG、图片和效果图层；
- 所属容器和层级；
- 裁切归属和 overflow 行为；
- 是否凸出所属容器；
- 是否参与组件最外轮廓；
- 透明合成分类；
- 组合轮廓、开口，以及必须保持无描边的内部接缝。

最外轮廓必须按所有成员的并集理解。日历、角标、页签、云朵、人物或插画只要凸出容器，就会改变整个组件轮廓。

## 图形方式选择

- 连续材质内部使用 `fill`。
- 源图明确存在的线使用 `stroke`。
- 多个重叠成员共用外沿时使用 `outer_ring`：先求轮廓成员并集，再向外扩张，最后减去原并集。
- 只有源图证明是柔和投影时才使用 `shadow`，不能用阴影代替精确复杂轮廓。

内部边缘需要消失时，禁止分别膨胀重叠成员。禁止为了连接两条外边而填死凹口。除非源图明确存在硬切口，否则禁止使用矩形硬裁切结束曲线或半透明外沿。

## 裁切和凸出检查

每个凸出图层都要执行：

1. 比较它和所属容器的源坐标框。
2. 登记实际裁切归属。
3. 确认 `overflow: hidden` 没有裁掉必需像素。
4. 把外沿放到裁切上下文之外，或者把凸出图层加入外轮廓并集。
5. 在 `AUDIT_MAP.json` 登记凸出区域和连接点。

## 透明证明

只有背景属于目标或背景已知时，才能直接用原始合成截图做普通像素比较。还要把候选组件分别放到白色、深蓝色和中性灰背景上截图，证明 DOM/SVG 半透明效果是真的，没有烘焙背景。使用紧贴组件的选择器，禁用动画，并给每张截图设置相同审计时间。这些截图只能证明实现属性，不能反推出未知的源 Alpha。
