# 真透明素材抠图

## 不可妥协规则

保留源 RGB，只创建或修正 Alpha 蒙版。模型重画、近似改色、不透明白底、色键底、棋盘格或网格都不是真正可复用的抠图。

## 操作步骤

1. 在 `ASSET_LEDGER.json` 中登记带留白的源图坐标框。
2. 选择蒙版来源：关键边缘优先手工蒙版；模型蒙版只作为初稿；只有几何条件合适时才使用 GrabCut、色键或椭圆。
3. 使用明确规格运行 `extract_alpha_assets.py`。
4. 对照源图和清单运行 `validate_alpha_assets.py`。
5. 以 100% 和放大比例检查白底、深蓝底合成图。
6. 修正头发、半透明材质、孔洞、边缘光晕和断开的组件，然后重新验证。

## 必须提供的证明

- 图片模式为 RGBA。
- 真前景抠图的 Alpha 同时包含 0 和 255。
- 需要透明的四角和留白边缘确实为 Alpha 0。
- 每个可见 RGB 像素都等于源图裁剪像素。
- 清单明确记录 `checkerboard_used: false`。
- 白底和深蓝底检查图存在。

`source_composited: true` 只能说明源截图的半透明像素已经混入背景色，不能把不透明背景当 Alpha 使用。如果前景 RGB、Alpha 和被遮挡背景都没有独立证据，必须把区域标记为 `non_identifiable`；Alpha 检查不能把一次合成观察变成唯一透明素材。
